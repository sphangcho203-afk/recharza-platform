# Recharza Social Presence

This document defines the official Recharza platform identity, account naming, profile copy, content purpose, and safety boundaries. The accounts may be reserved and prepared before public launch. No platform presence should imply that checkout, payment, or fulfilment is live before the website confirms it.

| Website | Recharza storefront | Pricing, checkout, tracking, terms, support entry | `recharza.com` |
| Gmail | Business mailbox | Account administration and early support | `recharza.store@gmail.com` |
| Telegram | Broadcast channel | Product updates, maintenance notices, offers, guides | `@recharza` |

## Content pillars

1. **Products:** supported games, packages, regional compatibility, availability
2. **Guides:** player IDs, region selection, checkout, order tracking
3. **Trust:** system status, payment safety, scam warnings, policy explanations
4. **Progress:** development updates and feature demonstrations
5. **Community:** gaming moments, polls, non-misleading promotions, player education

## Publishing rules

- Use the same name, logo, and core bio everywhere.
- Link back to the website as the source of truth.
- Clearly label previews, beta features, indicative prices, unavailable products, and maintenance.
- Never invent testimonials, order counts, partnerships, delivery speed, discounts, or publisher approval.
- Never publish customer emails, player IDs, order tokens, payment details, supplier credentials, or internal dashboards.
- Do not announce a game as available until its real server-side order gate is enabled and tested.
- Do not use personal accounts as official support channels.

## Account reservation checklist

- Reserve the Gmail address first.
- Reserve `@recharza` on Telegram, Instagram, YouTube, and Facebook.
- Create the WhatsApp Channel with the same display name and profile image.
- Enable two-factor authentication on every account.
- Store recovery codes offline and restrict administrator access.
- Use unique passwords and a password manager.
- Record platform owner, recovery email, recovery phone, and access roles in a private credential register.
- Publish nothing until the profile image, bio, security settings, and official-links list are consistent.
